一、ADB 基础认知

1. 定义

ADB（Android Debug Bridge，安卓调试桥）是 Android SDK 内置的命令行工具，用于建立电脑与 Android 设备（真机 / 模拟器）的通信，实现设备控制、应用调试、文件传输等核心功能，是安卓开发与测试的必备工具。

2. 核心能力

  • 设备管理：检测设备连接状态、切换 USB / 无线连接模式。

  • 应用操作：安装 / 卸载应用、启动 / 停止应用、查看进程日志。

  • 文件传输：电脑与设备间双向传输文件 / 文件夹。

  • 系统调试：执行设备 Shell 命令、查看系统信息、调试应用性能。

  • 特殊功能：模拟按键 / 触屏、截图录屏、管理开发者选项。

3. 工作架构（C/S 模式）

ADB 由三部分组成，协同实现命令传递与执行：

  • 客户端（Client）：运行于电脑，负责发送命令（如终端输入 adb devices），常见客户端有终端、Android Studio Logcat。

  • 服务端（Server）：电脑后台进程，监听 5037 端口（默认），管理客户端与设备守护进程的通信。

  • 守护进程（Daemon，adbd）：运行于设备后台，接收并执行服务端转发的命令，与设备硬件交互。

二、ADB 安装与配置

1. 安装方式

方式 1：通过 Android Studio 安装（推荐开发者）

  1. 打开 Android Studio，进入 Settings → Languages & Frameworks → Android SDK。
  2. 在 SDK Tools 中勾选 Android SDK Build-Tools 和 Android SDK Platform-Tools（ADB 核心包），点击 Apply 下载。
  3. 默认安装路径：

  • Windows：C:\Users\<用户名>\AppData\Local\Android\Sdk\platform-tools

  • Mac/Linux：~/Library/Android/sdk/platform-tools

我这里的安装路径为E:\Android\Sdk\platform-tools

方式 2：单独下载 Platform-Tools（非开发场景）

  1. 访问 Android 官方 Platform-Tools 下载页，按系统下载压缩包。

  2. 解压到本地目录（如 Windows 解压至 D:\adb，Mac 至 ~/adb），解压后含 adb.exe（Windows）或 adb（Mac/Linux）。

2. 环境变量配置（全局识别 adb 命令）

Windows 配置

  1. 右键 此电脑 → 属性 → 高级系统设置 → 环境变量。

  2. 在 系统变量 → Path 中点击 编辑 → 新建，粘贴 Platform-Tools 路径（如 D:\adb\platform-tools）。

  3. 关闭终端后重新打开，输入 adb --version，显示版本信息即成功。

Mac/Linux 配置

  1. 打开终端，执行命令（替换为实际路径）：

  • Mac：echo 'export PATH=$PATH:~/Library/Android/sdk/platform-tools' >> ~/.bash_profile（Zsh 用户改 ~/.zshrc）

  • Linux：echo 'export PATH=$PATH:~/adb/platform-tools' >> ~/.bashrc

  1. 使配置生效：

  • Mac：source ~/.bash_profile（Zsh：source ~/.zshrc）

  • Linux：source ~/.bashrc

  1. 验证：adb --version 显示版本信息。

3. 设备调试准备

真机配置

  1. 设备进入 设置 → 关于手机，连续点击 版本号 7 次，开启 开发者选项。

  2. 返回设置，进入 开发者选项，开启 USB 调试、USB 安装（部分机型）、USB 调试（安全设置）（部分机型）。

  3. 用 USB 连接电脑与设备，设备弹出 允许 USB 调试吗？，勾选 始终允许 并确认。

  4. 终端输入 adb devices，显示设备序列号与 device 即连接成功。

模拟器配置

  1. 启动模拟器（如 Android Studio Emulator、夜神模拟器）。

  2. 进入模拟器 设置 → 开发者选项，开启 USB 调试（步骤同真机）。

  3. 终端输入 adb devices，显示模拟器序列号（如 emulator-5554）与 device 即成功。

3. Android 11+ 无线调试（无 USB 初始连接）

  1. 设备 设置 → 开发者选项 → 开启无线调试。

  2. 点击 无线调试 → 使用配对码配对设备，记录 IP、端口、配对码。

  3. 电脑终端执行配对：adb pair 设备IP:端口（如 adb pair ``192.168.1.100:45678），输入配对码。

  4. 执行 adb connect 设备IP:新端口（新端口从无线调试页面查看），完成连接。

五、常见问题与解决方案

1. 问题：adb devices 显示 unauthorized（未授权）

  • 原因：设备未允许电脑调试。

  • 解决：

  1. 重新插拔 USB，等待设备弹出授权弹窗并确认。

  2. 设备 开发者选项 → 删除已授权电脑，重新连接。

  3. 重启设备与 ADB 服务（adb kill-server && adb start-server）。

2. 问题：adb devices 无设备显示

  • 原因：USB 驱动未装、数据线故障、开发者选项未开。

  • 解决：

  1. 换原装数据线 / USB 端口（排除充电线）。

  2. Windows 安装驱动：设备管理器 → 未知设备 → 更新驱动，指定 SDK 的 extras/google/usb_driver；品牌设备需官方驱动。

  3. 确认开发者选项与 USB 调试已开启。

3. 问题：adb install 提示 “INSTALL_FAILED_INSUFFICIENT_STORAGE”

  • 原因：设备存储空间不足。

  • 解决：

  1. 清理设备无用文件 / 应用。

  2. 尝试安装到 SD 卡（adb install -s 本地APK路径，需应用支持）。

4. 问题：无线连接 adb connect 失败

  • 原因：不在同一局域网、IP / 端口错误、端口占用。

  • 解决：

  1. 确认设备与电脑连同一 WiFi。

  2. 重新查看设备 IP 与端口，确保正确。

  3. 检查端口占用（Windows：netstat -ano | findstr 5555；Mac/Linux：lsof -i :5555），关闭占用进程或换端口（adb tcpip 5556）。

六、ADB 命令速查表

功能分类 命令示例 说明
设备管理 adb devices 查看已连接设备
adb connect 192.168.1.100:5555 无线连接设备
adb kill-server && adb start-server 重启 ADB 服务
应用操作 adb install -r app.apk 覆盖安装应用
adb uninstall com.tencent.mm 卸载微信
adb shell am start -n 包名/Activity 启动应用
文件传输 adb push ~/test.txt /sdcard/ 电脑→设备传文件
adb pull /sdcard/test.txt ~/Desktop/ 设备→电脑传文件
日志查看 adb logcat *:E 查看 Error 日志
adb logcat > ~/log.txt 保存日志到文件
系统操作 adb shell screencap /sdcard/s.png 设备截图
adb shell screenrecord /sdcard/r.mp4 设备录屏
adb reboot 重启设备
模拟操作 adb shell input keyevent 26 模拟电源键
adb shell input tap 500 1000 模拟触屏点击
调试辅助 adb shell pm clear 包名 清除应用数据
adb shell dumpsys meminfo 包名 查看应用内存
