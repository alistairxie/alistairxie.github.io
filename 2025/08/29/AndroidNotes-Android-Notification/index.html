Android Notification

Notification（通知）是 Android 系统提供的跨页面消息提醒组件，用于在应用后台或前台时，向用户推送重要信息（如消息通知、下载进度、日程提醒）。它可显示在状态栏、下拉通知栏，支持点击跳转、交互操作（如 “回复”“取消”），是提升用户粘性的核心工具。本文所有代码采用 Java 编写，从基础使用、类型分类、进阶功能到权限适配展开，帮助你灵活应对各类通知场景。

一、基础认知：Notification 的核心特性与分类

1.1 核心特性

  • 跨页面性：无论应用处于前台、后台还是关闭状态，均可触发通知（需通过 Service、WorkManager 等后台组件）；

  • 可交互性：支持设置点击事件（跳转 Activity/Fragment）、添加操作按钮（如 “回复消息”“查看详情”）；

  • 多形态展示：支持普通文本通知、折叠式通知（含大图 / 长文本）、悬浮式通知（Android 5.0+）、进度条通知等；

  • 权限可控：Android 13+（API 33+）需动态申请POST_NOTIFICATIONS权限，用户可在系统设置中关闭应用通知权限。

1.2 常见分类（按功能与形态）

类型 核心用途 特点 适用场景
普通文本通知 简单信息提醒（如 “收到一条消息”） 仅含标题、内容、小图标，结构简单 短信通知、系统提示、简单状态更新
折叠式通知 展示长文本 / 大图（如新闻摘要、商品图片） 默认折叠显示，下拉展开后显示完整内容（需自定义 RemoteViews） 新闻推送、商品推荐、长内容通知
进度条通知 展示任务进度（如文件下载、数据同步） 含水平 / 圆形进度条，支持实时更新进度值 下载管理、备份进度、视频转码
悬浮式通知 前台重要提醒（如来电、视频通话） 直接显示在屏幕顶部，无需下拉通知栏即可操作（需开启前台优先级） 来电提醒、视频通话邀请、即时消息
媒体控制通知 控制后台媒体播放（如音乐、视频） 含播放 / 暂停、上一曲 / 下一曲按钮，支持媒体信息展示 音乐播放器、视频播放器

  • 通知渠道重要性（IMPORTANCE）决定通知的显示优先级，共 5 级：

    • IMPORTANCE_NONE（0）：无通知，不显示；

    • IMPORTANCE_MIN（1）：仅显示在下拉通知栏，无声音 / 震动；

    • IMPORTANCE_LOW（2）：显示在状态栏，无声音，下拉可看到；

    • IMPORTANCE_DEFAULT（3）：默认级别，显示在状态栏，有声音，无悬浮；

    • IMPORTANCE_HIGH（4）：显示在状态栏，有声音 + 悬浮（Android 12 + 需额外设置setForegroundServiceBehavior）。

3.3 悬浮式通知（Android 5.0+）

悬浮式通知（Heads-Up Notification）直接显示在屏幕顶部，无需下拉通知栏即可操作，适合重要提醒（如来电、视频通话）。需满足两个条件：

  1. 通知渠道重要性设为IMPORTANCE_HIGH（Android 8.0+）或优先级设为PRIORITY_HIGH（Android 7.1 及以下）；

  2. 应用处于前台或后台（非关闭状态）。

  • addAction最多添加 3 个操作按钮，图标建议使用单色小图标，避免显示异常。

四、避坑指南：Notification 常见问题与解决方案

问题 1：通知不显示（Android 8.0+）

  • 原因 1：未创建通知渠道，或渠道 ID 与 Builder 中设置的渠道 ID 不一致；

  • 解决：确保createNotificationChannel()已调用，且Builder的第二个参数（渠道 ID）与创建的渠道 ID 完全一致（如CHANNEL_ID_NORMAL）。

  • 原因 2：通知权限未授权（Android 13+）或用户在系统设置中关闭了通知；

  • 解决：通过checkSelfPermission检查权限，未授权时发起请求；用户关闭时，引导调用openNotificationSettings()跳转到设置页开启。

  • 原因 3：setSmallIcon未设置或图标格式错误（如非透明背景的彩色图标）；

  • 解决：必须设置setSmallIcon，且使用透明背景的单色图标（如白色），可放在mipmap-xxxhdpi目录下。

问题 2：PendingIntent 跳转无响应或参数传递失败

  • 原因 1：PendingIntent的requestCode重复，导致 Intent 被覆盖；

  • 解决：不同PendingIntent使用不同的requestCode（如接听按钮用 1，挂断按钮用 2）。

  • 原因 2：PendingIntent未设置FLAG_UPDATE_CURRENT，导致 extra 数据未更新；

  • 解决：创建PendingIntent时添加PendingIntent.FLAG_UPDATE_CURRENT，确保参数实时更新。

  • 原因 3：Android 12 + 未添加FLAG_IMMUTABLE，导致PendingIntent创建失败；

  • 解决：按版本分支添加FLAG_IMMUTABLE（参考 2.4 节代码）。

问题 3：通知图标显示异常（如显示默认图标而非自定义图标）

  • 原因：setSmallIcon使用了彩色图标或非透明背景图标，在深色状态栏中被系统替换为默认图标；

  • 解决：使用透明背景的单色图标（如白色），可通过 Android Studio 的 “Image Asset” 工具生成符合规范的通知图标（路径：File→New→Image Asset→Icon Type 选择 Notification Icons）。

问题 4：悬浮通知不显示（Android 5.0+）

  • 原因 1：通知优先级 / 重要性不足（如IMPORTANCE_DEFAULT或PRIORITY_DEFAULT）；

  • 解决：Android 8.0 + 将渠道重要性设为IMPORTANCE_HIGH，Android 7.1 及以下将Builder的优先级设为PRIORITY_HIGH。

  • 原因 2：Android 12 + 应用后台时，未使用前台服务或未设置setForegroundServiceBehavior；

  • 解决：后台悬浮需启动前台服务，并在通知中设置：

问题 5：进度条通知更新后进度不刷新

  • 原因：每次更新进度时未重新调用notificationManager.notify()，或使用了不同的通知 ID；

  • 解决：每次更新进度后，使用相同的通知 ID（如 103）调用notify()，确保覆盖原有通知。
