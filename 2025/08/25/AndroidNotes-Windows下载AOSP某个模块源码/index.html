Windows 下载 AOSP 某个模块源码

在安卓开发中，有时无需下载完整 AOSP 源码（需 100GB+），仅需获取单个模块（如frameworks/native）进行学习或定制。本文将整合在线源码查看工具与高效下载方法，重点补充国内镜像适配、分支管理等关键细节，适配 Windows 系统操作场景。

一、常用 AOSP 在线源码查看网站（高效查路径 / 版本）

在线查看源码可先确认模块路径、分支信息，避免盲目下载。以下是 3 个主流平台的对比与适用场景：

网站地址 支持 Android 版本 核心优势 局限性 适用场景
androidxref.com Android 1 ~ 9 界面简洁，支持按路径 / 关键词搜索，可查看文件依赖关系 最高仅支持 Android 9，无新版本源码 查阅旧版本系统框架（如 Android 7/8）
aospxref.com Android 5 ~ 15 版本覆盖全（含最新 Android 15），支持分支切换、代码高亮 部分冷门模块索引较慢，无官方技术支持 日常查新 / 旧版本模块源码（推荐）
cs.android.com 全版本（含预览版） 谷歌官方平台，更新实时，支持关联 issue、文档跳转 国内访问需代理，加载速度受网络影响较大 查阅官方最新源码、确认模块官方路径

关键使用技巧

  1. 快速定位模块路径：在aospxref.com首页搜索框输入模块功能（如 “Input”），或直接输入已知路径（如frameworks/native），即可查看模块下的文件结构；

  2. 确认分支对应关系：在cs.android.com中，每个模块页面顶部会显示 “Branch” 下拉框，可查看该模块在不同 Android 版本（如android-14.0.0_r15）中的代码差异。

二、Windows 下载 AOSP 单个模块源码

下载核心逻辑：先获取模块的 Git 仓库地址→再通过 Git 克隆指定模块（支持国内镜像、指定分支），无需依赖 Repo 工具（简化操作，适合仅下载单个模块的场景）。

2.1 前置准备：安装 Git（Windows 必备）

  1. 下载安装：前往GitforWindows，选择 64 位安装包（如Git-2.47.0-64-bit.exe）；

  2. 关键安装选项：

  • 勾选 “Add Git Bash Here”（右键菜单添加 Git Bash，方便后续在任意目录操作）；

  • 勾选 “Use Git from Git Bash only”（避免与系统其他工具冲突）；

  1. 验证安装：右键空白处打开 “Git Bash”，输入git --version，输出类似 “git version 2.47.0.windows.1” 即成功。

2.2 步骤 1：获取模块的 Git 仓库地址（2 种方法）

需先确认目标模块的官方 Git 仓库地址（格式：host + 模块name + .git），其中host可替换为国内镜像地址（解决访问慢问题）。

2.3 步骤 2：替换为国内镜像地址

国内直接访问谷歌android.googlesource.com速度极慢，需将host替换为国内镜像的host，常用镜像对应关系如下：

中科大镜像 - https://mirrors.ustc.edu.cn/git/AOSP/ https://mirrors.ustc.edu.cn/git/AOSP/platform/frameworks/native.git

2.5 步骤 4：验证下载结果

进入下载后的模块目录（如D:\AOSP_Modules\frameworks\native），查看是否包含核心文件：

  • 若存在src（源码目录）、Android.mk/Android.bp（编译配置文件），则表示下载完整；

  • 若提示 “fatal: repository not found”，需检查 Git 仓库地址是否拼写正确（尤其是name部分，如是否多写 / 少写 “platform/”）。

问题 2：国内镜像克隆提示 “404 Not Found”

  • 原因：模块的name拼写错误，或该镜像未同步该模块；

  • 解决：

  1. 核对name（参考default.xml或cs.android.com确认正确路径）；

  2. 更换镜像（如清华镜像提示 404，可尝试中科大镜像）。

四、两种下载方式对比（Repo vs 直接 Git 克隆）

下载方式 依赖工具 优势 劣势 适用场景
本文方法（Git 克隆） Git 操作简单，无需 Repo，仅下载单个模块，节省空间 无法批量下载多个模块，需手动管理依赖 仅需下载 1~2 个模块（如学习、修改单个模块）
Repo 同步 Git+Repo 支持批量同步多个模块，自动处理依赖关系 需初始化 Repo 清单，操作较复杂，占用空间大 下载多个关联模块（如frameworks/base+packages/apps/Settings）
