一、SharedPreferences 概述

1. 定义

SharedPreferences（简称 SP）是 Android 系统提供的一种轻量级数据存储方式，本质是基于 XML 文件存储键值对（Key-Value）数据的工具类，主要用于存储少量、简单的配置信息，如用户登录状态、应用个性化设置（字体大小、主题颜色）等。

2. 核心特点

  • 轻量级：存储数据量有限（建议不超过 100KB），不适合存储大量或复杂数据（如图片、大文本）。

  • 键值对结构：数据以 “键 = 值” 形式存储，键为 String 类型，值支持 String、Int、Long、Float、Boolean 五种基本数据类型。

  • XML 存储：数据最终保存为 XML 文件，默认存储路径为/data/data/<应用包名>/shared_prefs/，文件名可自定义。

  • 线程安全：SharedPreferences 的读取操作是线程安全的（内部通过锁机制实现），但写入操作需注意同步问题。

  • 持久化存储：数据会长期保存，除非手动清除（如卸载应用、清除应用数据）或代码删除。

三、存储模式说明（重要）

初始化 SP 时的 “存储模式” 参数决定了 SP 文件的访问权限，Android 不同版本对模式的支持不同，需特别注意兼容性：

模式常量 作用说明 适用版本 注意事项
MODE_PRIVATE 仅当前应用可访问（私有模式） 所有版本（推荐） 最安全的模式，其他应用无法读取或修改
MODE_WORLD_READABLE 允许其他应用读取当前 SP 文件 Android 1.0 - 6.0 Android 7.0（API 24）后废弃，存在安全风险
MODE_WORLD_WRITEABLE 允许其他应用修改当前 SP 文件 Android 1.0 - 6.0 Android 7.0 后废弃，严重安全隐患
MODE_MULTI_PROCESS 支持多进程访问（已废弃） Android 1.0 - 2.3 Android 3.0 后不推荐，多进程需用 ContentProvider

注意：Android 7.0 及以上版本，若使用MODE_WORLD_READABLE或MODE_WORLD_WRITEABLE，会直接抛出SecurityException，因此所有场景优先使用MODE_PRIVATE。

四、适用场景与不适用场景

1. 适用场景

  • 存储应用配置：如是否开启推送、主题设置、字体大小等。

  • 存储用户轻量信息：如用户名、登录状态（是否记住密码）。

  • 存储临时标记：如首次启动引导标记、是否已展示广告等。

2. 不适用场景

  • 存储大量数据：如列表数据、大文本（SP 读取效率低，易导致 ANR）。

  • 存储复杂数据：如对象、集合（需手动序列化，操作繁琐）。

  • 存储敏感数据：如密码、银行卡号（SP 文件为明文存储，安全性差，需加密）。

  • 多进程共享数据：SP 不支持跨进程同步，多进程场景需用ContentProvider或MMKV。

五、注意事项与优化建议

1. 安全性问题

  • 明文存储风险：SP 的 XML 文件为明文格式，直接存储敏感数据（如密码）会被轻易破解。解决方案：

  1. 对敏感数据进行加密（如 AES 加密）后再存储。

  2. 使用 AndroidX 提供的EncryptedSharedPreferences（推荐，封装了加密逻辑）。

  • 权限控制：务必使用MODE_PRIVATE，避免使用已废弃的公开模式。

2. 性能优化

  • 减少频繁读写：避免在循环、UI 线程中频繁调用apply()或commit()（尤其是commit()会阻塞线程）。

  • 批量操作：多个键值对的写入 / 修改应合并为一次 Editor 操作，减少 IO 次数。

  • 避免存储大数据：单个 SP 文件建议不超过 100KB，大数据推荐用Room（数据库）或File存储。

3. 兼容性问题

  • Android 7.0 + 模式废弃：不再支持MODE_WORLD_READABLE和MODE_WORLD_WRITEABLE，需替换为MODE_PRIVATE。

  • 多进程问题：若应用需多进程共享数据，SP 无法满足需求，推荐使用腾讯的MMKV（基于 mmap 的高效存储框架，支持多进程、加密）。

4. 其他注意事项

  • 键的唯一性：同一 SP 文件中，键名需唯一，否则会出现值覆盖问题。

  • 默认值设置：读取数据时务必指定合理的默认值（如字符串默认值用""，而非null，避免空指针异常）。

  • 清理无用数据：当某些键值对不再使用时，应及时调用remove()删除，避免冗余数据占用空间。

六、替代方案推荐

当 SP 无法满足需求时（如多进程、加密、大量数据），可考虑以下替代方案：

  1. MMKV：腾讯开源的高效存储框架，基于 mmap，支持键值对、加密、多进程，性能远超 SP。

  2. DataStore：Google 推荐的 SP 替代方案，基于 Kotlin 协程，支持异步、类型安全，分为Preferences DataStore（键值对）和Proto DataStore（对象存储）。

  3. Room：Android 官方 ORM 数据库框架，适合存储大量、复杂结构的数据。
